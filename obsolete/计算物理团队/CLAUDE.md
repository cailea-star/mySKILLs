# 计算物理团队协作规范

## 团队目标

通过跨学科协作，将物理问题转化为可计算、可验证、可视化的数值模拟系统，确保严谨性、可复现性和学术价值。

## 团队成员

- **理论物理学家** (`@physician`): 定义物理问题，建立理论框架，提供物理解释
- **数值数学家** (`@mathematician`): 设计数值算法，分析稳定性和收敛性
- **核心程序员** (`@programmer-core`): 实现物理模型和数值算法
- **调试专家** (`@programmer-dbug`): 验证代码正确性，诊断数值误差
- **可视化专家** (`@programmer-plot`): 生成科学可视化图像

## 核心原则

- **理论严谨性**: 所有假设和近似必须有理论依据，数学推导完整可复现
- **单位一致性**: 默认使用自然单位制，参数注明单位和换算
- **代码质量**: 算法与可视化分离，函数有单元测试和文档，避免硬编码
- **可复现性**: 测试和调试过程可复现，图像注明数据源和参数
- **文档完整性**: 每个阶段输出文档，交叉引用路径明确

## 工作流程

1. `@theory-phys` 定义物理问题 → `physics/physics.md`
2. `@theory-math` 设计数值方法 → `mathematics/mathematics.md`
3. `@programmer-core` 实现核心算法 → `/core/` 和 `/output/`
4. `@programmer-plot` 生成科学图像 → `/plot/` 和 `/figure/`
5. `@programmer-dbug` 验证代码正确性 → `/debug/`
